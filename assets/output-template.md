# Output Template

## 1. Product Overview

- Product name
- Current stage
- Target users
- Problem being solved

## 2. Goals and Scope

- Goals
- Scope
- Non-goals
- Success criteria

## 3. User Roles

- Role list
- Responsibilities
- Permission boundaries

## 4. Core Workflows

- Role-specific workflow for each core role
- Trigger, main actions, system actions, result, end condition

## 5. Data Sources and Field Design

- Source list
- Source acquisition method
- Raw-to-structured mapping
- Field list
- Field purpose

## 6. End-to-End System Flow

- Full business and system chain

## 7. Functional Modules

- Module list
- Module objective
- Responsibilities

## 8. State Design

- State definitions
- Transition conditions
- State update rules

## 9. Non-Functional Requirements

- Performance
- Reliability
- Traceability
- Security or permissions if needed

## 10. Acceptance Criteria

- Verifiable completion conditions

## 11. Risks and Open Questions

- Risks
- Assumptions
- Pending confirmations

## 12. Development Handoff Notes

- Suggested module split
- Suggested implementation order
- Key implementation cautions
